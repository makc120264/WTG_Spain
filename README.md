# ТЗ Socket
Необхідно реалізувати базову систему реєстрації та авторизації юзерів.
Доступ до функціоналу повинен бути лише для авторизованих юзерів.
Після входу в систему юзер повинен бачити список усіх зареєстрованих юзерів.
Для кожного юзера повинна бути можливість відправити повідомлення з подальшим зберіганням у базі даних:
- відправник
- отримувач
- текст повідомлення
- час створення

Після авторизації кожен юзер повинен підключатися до WebSocket та слухати події, адресовані саме йому.
Логіка роботи:
1. Юзер відправляє повідомлення.
2. Повідомлення зберігається у базі даних.
3. Сервер генерує подію.
4. Подія відправляється через WebSocket.
5. Якщо подія адресована конкретному юзеру, він отримує її у браузері.
6. Якщо отримувач був офлайн, то під час наступного входу в систему він повинен побачити пропущені для нього повідомлення.
7. Повідомлення повинні відображатися без перезавантаження сторінки.

Код повинен бути організований з використанням Events та Broadcasting.


Реалізовано проект на Laravel 11 + PHP 8.3 з базовою системою:

- реєстрація/авторизація користувачів (Laravel Breeze);
- Доступ до основного функціоналу тільки для авторизованих;
- Список всіх зареєстрованих користувачів після входу;
- Надсилання повідомлень між користувачами;
- зберігання повідомлень у БД (`sender_id`, `recipient_id`, `body`, `created_at`);
- адресна доставка подій через WebSocket (Laravel Reverb+Redis);
- Відображення нових повідомлень без перезавантаження сторінки;
- Показ пропущених вхідних повідомлень при наступному вході (завантаження з БД на dashboard).

## Встановлення

1. Встановити залежність:

``` bash
composer install
npm install
````

2. Налаштувати оточення:

``` bash
cp .env.example .env
php artisan key:generate
````

3. Заповнити `.env` (MySQL, Redis, Reverb):

``env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=wtg_spain
DB_USERNAME=root
DB_PASSWORD=

BROADCAST_CONNECTION=reverb
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
````

4. Виконати міграції та зібрати фронт:

``` bash
php artisan migrate
npm run build
````

## Запуск

В окремих терміналах:

``` bash
php artisan serve
php artisan reverb:start
npm run dev
````

## Основний потік

1. Користувач реєструється/логіниться.
2. Попадає на `/dashboard`.
3. Бачить список користувачів та форму надсилання.
4. Відправляє повідомлення - воно зберігається в БД і broadcast-іє подією `MessageSent`.
5. Отримувач слухає приватний канал `users.{id}` та отримує нові повідомлення в реальному часі.
